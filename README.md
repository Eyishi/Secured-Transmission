# Secured-Transmission
该项目是对网络通信提供加解密的模块，基于protobuf对数据进行序列化；openssl对数据进行加解密；外联接口对密钥进行操作。
秘钥协商
- 秘钥协商服务器
- 密钥协商客户端
  - 这两部分不会部署在同一台主机上
- 知识点:
  - 套接字通信
    - tcp
      - 粘包问题
      - 阻塞的超时处理
        - 使用IO转接函数
      - 类的封装
  - 数据的跨平台传输
    - 数据的序列化
      - protobuf
      - json
  - 设计模式
    - 工厂模式
    - 简单工厂模式
      - 创建对象, 实现了多态的子类对象
    - 单例模式
      - 对象永远只有一个
  - 配置文件 -> json
    - 解析: 使用jsoncpp
      - Value
      - Reader
      - FastWriter
  - 秘钥交换(秘钥分发) -> RSA
    - 有公钥的一方生成一个随机字符串 -> 对称加密的秘钥
    - 使用公钥对称对称加密的秘钥加密 -> 密文
    - 将密文发送到有私钥的一方
    - 有私钥的一方使用私钥解密 -> 对称加密的秘钥
  - 数据签名 -> RSA
    - 目的: 
      - 检测数据有没有被篡改(数据的完整性)
      - 校验数据的所有者
    - 签名:
      - 对原始数据进行哈希原始 -> 哈希值
      - 使用私钥对哈希值加密 -> 密文
      - 将密文和原始数据发送出去 -> 等待那公钥的人校验
    - 校验签名:
      - 接收数据: 密文 + 原始数据
      - 对原始数据进行哈希原始 => 哈希值2
      - 使用公钥对密文解密 -> 哈希值1
      - 比较两个哈希值
  - 哈希算法
    - md5
    - sha1
    - sha224, sha256, sha384, sha512
